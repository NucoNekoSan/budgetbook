# BudgetBook

Django + HTMX で作ったシンプルな家計簿アプリです。  
月次ダッシュボード、年間サマリー、収支登録、カテゴリ別集計、口座残高の確認、口座・カテゴリ管理を最小構成で入れています。

## 主な機能

- 月別の収入・支出・収支差額の表示（差額は黒字=緑 / 赤字=赤 で色分け）
- 取引（収支）の追加 / 編集 / 削除（HTMX による部分更新）
- 種別（収入 / 支出）に連動したカテゴリ絞り込み
- 取引一覧の摘要検索・口座フィルタ・カテゴリフィルタ
- 口座ごとの残高表示
- 支出カテゴリ別の集計
- 取引一覧のページネーション（20 件/ページ）
- 月次取引の CSV エクスポート（`kakeibo-YYYY-MM.csv`、UTF-8 BOM 付き）
- 月次推移チャート（直近 12 か月の収入・支出・差額を棒＋折れ線で表示、Chart.js）
- 年間サマリー（`/annual/`）— 年ナビゲーション、年間サマリーカード、月別テーブル、年間推移チャート
- 操作後フラッシュメッセージの自動消去（CSS アニメーション、約 4 秒）
- 設定画面（`/settings/`）から口座・カテゴリの一覧 / 追加 / 編集 / 有効・無効切り替えが可能
- 初期カテゴリを一括投入する `seed_budget_data` コマンド付き
- ログイン認証（Django 標準認証、セッションベース）
- Django TestCase による自動テスト（61 件）
- CSS は `static/css/style.css` に分離（Django の `{% static %}` 経由で読み込み）

## 前提

- Python 3.10 以上
- SQLite を使用（初期設定）

## セットアップ

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
source .venv/bin/activate

python -m pip install -U pip
python -m pip install -r requirements.txt

# 環境変数の設定（初回のみ）
cp .env.example .env
# .env を開いて SECRET_KEY にランダムな文字列を設定してください
# 例: python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

python manage.py migrate
python manage.py seed_budget_data
python manage.py runserver
```

> **注意**: `.env` ファイルがないと起動時にエラーになります。必ず `.env.example` をコピーして作成してください。`.env` は Git 管理対象外です。

> **スマホや他端末からアクセスする場合**: `.env` の `ALLOWED_HOSTS` に PC の LAN IP を追加してください。例: `ALLOWED_HOSTS=127.0.0.1,localhost,192.168.1.10`。IP は `ipconfig`（Windows）や `ip addr`（Linux）で確認できます。サーバー起動時は `python manage.py runserver 0.0.0.0:8000` で全インターフェースをリッスンしてください。

### ユーザー作成

ログイン認証が必要です。初回セットアップ時にユーザーを作成してください。
夫婦 2 人で利用する場合は、管理者 1 名（superuser）＋一般ユーザー 1 名で十分です。

```bash
# 1人目（管理者）— superuser として作成
python manage.py createsuperuser

# 2人目（一般ユーザー）— Django shell で作成
python manage.py shell -c "
from django.contrib.auth.models import User
User.objects.create_user('wife', password='任意のパスワード')
"
```

> superuser は Django admin（`/admin/`）にもアクセスできます。一般ユーザーは家計簿アプリのみ利用可能です。家計簿の機能（取引登録・設定変更など）はどちらのユーザーでも同じように使えます。

ブラウザで以下を開きます。

- アプリ本体: `http://127.0.0.1:8000/`
- 年間サマリー: `http://127.0.0.1:8000/annual/`
- 設定画面: `http://127.0.0.1:8000/settings/`
- 管理画面: `http://127.0.0.1:8000/admin/`（`createsuperuser` 実行済みの場合のみ）

## 初期データ

`seed_budget_data` を実行すると、以下を作成します。

- 口座: 現金 / 普通預金
- 収入カテゴリ: 給与 / 副収入 / 臨時収入
- 支出カテゴリ: 食費 / 日用品 / 住居費 / 水道光熱費 / 通信費 / 交通費 / 医療費 / 娯楽費 / 教育費 / 雑費

必要に応じて設定画面（`/settings/`）から変更してください。

## 画面構成

### ダッシュボード（`/`）

- 左: 月次サマリー、支出カテゴリ集計、月次推移チャート、口座残高、取引一覧（検索・フィルタ付き）
- 右: 収支入力フォーム

### 年間サマリー（`/annual/`）

- 年間の収入・支出・差額をサマリーカードで表示
- 月別テーブルで 1〜12 月の内訳を一覧（各月リンクからダッシュボードへ遷移）
- 年間推移チャート（月ごとの収入・支出を棒グラフ、差額を折れ線で表示）
- 「← 前年」「次年 →」で年を切り替え（未来年は現在年にクランプ）

### 設定（`/settings/`）

- 口座の一覧 / 追加 / 編集 / 有効・無効切り替え
- カテゴリの一覧 / 追加 / 編集 / 有効・無効切り替え
- 変更はダッシュボードの取引フォームにすぐ反映される

## バックアップと復元

この家計簿アプリのデータはローカルの 2 ファイルに集約されています。

| ファイル | 内容 | バックアップ方法 |
|----------|------|----------------|
| `db.sqlite3` | 全取引・口座・カテゴリなどのデータベース | 日常バックアップ対象 |
| `.env` | SECRET_KEY などの環境設定 | 変更時に別途安全な場所に保管 |

### 手動バックアップ（PowerShell）

`runserver` を停止した状態で、以下を実行します。

```powershell
$dir = "backup\$(Get-Date -Format 'yyyy-MM-dd_HHmmss')"
New-Item -ItemType Directory -Path $dir | Out-Null
Copy-Item db.sqlite3 -Destination $dir
Write-Host "バックアップ完了: $dir"
```

`backup/` ディレクトリにタイムスタンプ付きフォルダが作られ、データベースがコピーされます。

### 復元（PowerShell）

> **必ず `python manage.py runserver` を停止してから実行してください。** SQLite はサーバー実行中にコピーするとデータが破損する可能性があります。

復元は上書きになります。現在のデータを残したい場合は、先に上のバックアップ手順で退避してから実行してください。

```powershell
# 例: 2025-04-18_120000 のバックアップから復元する場合
Copy-Item backup\2025-04-18_120000\db.sqlite3 -Destination . -Force
```

復元後、`python manage.py runserver` で起動し、データが正しく戻っていることを確認してください。

### .env の保管について

`.env` は SECRET_KEY などの秘密情報を含むため、日常バックアップとは分けて管理してください。

- `.env` を変更した際は、USBメモリやパスワードマネージャーなど **第三者がアクセスできない安全な場所** に別途保管してください
- 共有フォルダ・クラウド同期フォルダ・公開リポジトリには置かないでください
- SECRET_KEY が外部に露出した可能性がある場合は、以下のコマンドで再生成してください（再生成後は全ユーザーの再ログインが必要です）

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### 注意事項

- `backup/` は `.gitignore` に含まれているため、Git には追跡されません。
- バックアップの定期実行が必要な場合は、Windows のタスクスケジューラに上記スクリプトを登録することもできます。

## 既知の仕様・注意点

- **取引一覧は 20 件/ページ** で表示されます。それ以上の取引はページネーションで参照できます。
- **保存・編集・削除後は 1 ページ目に戻ります**。操作後に最新の取引が先頭に表示される仕様です。
- **CSV エクスポートはページに関わらず、表示中の月の全件を出力します。**
- **口座の初期残高は作成後に変更できません。** 初期残高は口座残高の計算基準になるため、編集画面では読み取り専用です。修正が必要な場合は新しい口座を作成してください。
- **カテゴリの区分（収入 / 支出）は作成後に変更できません。** 既存取引との整合性を保つためです。区分を変えたい場合は新しいカテゴリを作成し、旧カテゴリを無効にしてください。
- **口座・カテゴリの削除はできません。** 取引が紐づいている可能性があるため、使わなくなった口座・カテゴリは「無効」に切り替えてください。無効にすると取引フォームの選択肢から非表示になりますが、既存取引のデータは保持されます。
- 金額フィールドは 1 以上の整数のみ受け付けます。
- **現在の認証は家庭内・私的利用向けの最小構成です。** 外部（インターネット）に公開する場合は、HTTPS の導入と `SESSION_COOKIE_SECURE = True` / `CSRF_COOKIE_SECURE = True` の設定が必要です。
